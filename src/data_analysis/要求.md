# Role
你是一位精通電腦視覺 (Computer Vision) 與高維資料分析的資深 Python 架構師。你熟悉最新的 Deep Learning 框架 (PyTorch, Sentence-Transformers) 以及流形學習演算法 (Manifold Learning)。

# Objective
請開發一個端到端 (End-to-End) 的 Python 類別 (Class) 或模組，用於分析工程 CAD 圖檔 (PDF 格式)。
核心任務是：將 PDF 轉換為圖像 -> 使用 SOTA 的 OpenCLIP 模型提取特徵向量 (Embedding) -> 使用 UMAP 與 t-SNE 進行降維 -> 產生互動式的視覺化散佈圖。

# Technical Constraints & Stack
1.  **PDF 處理**: 使用 `PyMuPDF` (fitz) 將 PDF 轉換為高解析度圖片。
2.  **特徵提取 (Core)**: 必須使用 `sentence-transformers` 庫載入 OpenCLIP 模型。
    * **模型指定**: 預設使用在 LAION-2B 數據集上訓練的 SOTA 模型 (例如 `laion/CLIP-ViT-L-14-laion2B-s32B-b82K` 或同等級別)，以確保對幾何線條與非自然圖像的最佳語義理解。
    * **運算**: 支援 GPU (CUDA/MPS) 加速，並實作 **Batch Processing** 以處理大量圖檔。
3.  **降維演算法**:
    * **UMAP**: 使用 `umap-learn` 庫。參數須針對 Cosine Distance 進行優化。
    * **t-SNE**: 使用 `scikit-learn` 或 `openTSNE`。
4.  **視覺化**: 使用 `plotly` 產生**互動式 (Interactive)** 散佈圖，滑鼠懸停 (Hover) 時應顯示檔名與類別。
5.  **資料結構**: 使用 `pandas` DataFrame 管理檔案路徑、標籤與向量。

# Functional Requirements (Step-by-Step Implementation)

## 1. Data Loading & Preprocessing
* 輸入為一個包含子資料夾的根目錄，子資料夾名稱即為「類別標籤 (Label)」。
* 資料夾內為 PDF 文件，因此需要先將 PDF 轉成 PNG。
* 實作一個快取機制 (Caching)，若偵測到已轉換的圖片或已提取的 `.npy` 向量檔，則跳過重複運算。

## 2. Feature Extraction (The "OpenCLIP" Engine)
* 實作 `OpenCLIPExtractor` 類別。
* 輸入圖片前需進行標準的 CLIP Preprocessing (Resize, Normalize)。
* 輸出應為 Normalize 過的 Numpy Array (Shape: `[N_samples, Embedding_Dim]`)。

## 3. Dimensionality Reduction
* 實作 `ManifoldReducer` 類別。
* **關鍵設定**: 針對 OpenCLIP 的向量特性，UMAP 的 `metric` 必須設為 `'cosine'` 而非 `'euclidean'`。
* 提供參數讓使用者調整 `n_neighbors` (控制局部/全域結構) 與 `min_dist`。

## 4. Visualization
* 產生兩個圖表：UMAP Projection 與 t-SNE Projection。
* 圖表須美觀、專業，不同類別使用高對比度顏色區分。
* 圖表標題需註明使用的模型名稱與樣本數。
